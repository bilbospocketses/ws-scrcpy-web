Subnet & CIDR Cheat Sheet

A plain-English guide to the subnet formats ws-scrcpy-web accepts.

What is a subnet?

An IP address is split into two parts: one part identifies the network, the other identifies a specific host (device) on that network. A subnet is a named range of addresses that all share the same network part.

On a home network, every device you own — phone, laptop, TV, router — is on the same subnet. Scanning a subnet means checking every possible host address in that range.

What is CIDR notation?

CIDR (Classless Inter-Domain Routing) writes a subnet as an address plus a slash number, like 192.168.1.0/24. The number after the slash is the prefix length — how many bits (out of 32) belong to the network part.

The smaller the prefix, the bigger the subnet. /24 holds 256 addresses (254 usable); /16 holds 65,536 addresses.

Private IP ranges (RFC 1918)

Home and office networks use one of three IP address ranges that are reserved for private use. Your router picks one and hands out addresses inside it to every device.

  • 10.0.0.0/8 — 16,777,216 addresses. Often used in large organizations.
  • 172.16.0.0/12 — 1,048,576 addresses. Less common.
  • 192.168.0.0/16 — 65,536 addresses. The default for most home routers.

What does my home router use?

Most consumer routers (Netgear, TP-Link, Linksys, Google Wifi, etc.) default to either 192.168.0.0/24 or 192.168.1.0/24. Some ISPs (including Xfinity and Verizon) use 192.168.86.0/24 or similar.

To check your own subnet:

  • Windows: open Command Prompt, run ipconfig. Look for "IPv4 Address" and "Subnet Mask" under your Wi-Fi or Ethernet adapter.
  • Mac / Linux: open Terminal, run ifconfig or ip addr. Look for an inet line showing your address and /N prefix.

If your IPv4 address is 192.168.1.42 with netmask 255.255.255.0, you're on 192.168.1.0/24.

How to enter subnets in ws-scrcpy-web

The "add subnet" dialog accepts three formats:

1. CIDR notation — the canonical form. Examples:

192.168.1.0/24
10.0.0.0/16
192.168.86.0/24

Prefixes from /16 to /32 are allowed. Larger prefixes (/15 and below) are rejected because scanning more than 65,534 hosts at once is impractically slow. If you need to cover more, add multiple /16 entries.

2. Single IP — treated as /32 (one host). Useful when you know exactly one device's address:

192.168.1.5

3. IP range — a contiguous block within a single /24. Two forms:

192.168.1.10-192.168.1.50
192.168.1.10-50

Both mean "addresses .10 through .50 on the 192.168.1.x network" — 41 hosts. Ranges must stay within the same /24 (only the last number changes). For wider ranges, use CIDR.

What does the scan actually do?

For each host in the subnets you've listed, ws-scrcpy-web does two things in parallel:

  • mDNS discovery — asks the local network which devices advertise the ADB service. Modern Android devices with wireless debugging enabled usually show up here.
  • TCP port-5555 probe — tries to open a TCP connection to port 5555 on each host. If it succeeds, the scanner runs adb connect to confirm the host actually speaks ADB (not just something else listening on that port), then disconnects cleanly and adds the host to your available devices list.

The scan is read-only: nothing is written to or changed on any device. It's equivalent to checking whether each address has a doorbell, without entering.
